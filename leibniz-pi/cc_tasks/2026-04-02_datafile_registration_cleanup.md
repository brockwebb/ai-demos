# CC Task: DataFile Registration Cleanup

**Date:** 2026-04-02
**Purpose:** Promote all DataFile artifacts from `proposed` to `verified` state, backfilling documentation properties where missing.

---

## Pre-Flight

1. Read `paper/glossary.md` and `paper/conventions.md`
2. Run `seldon artifact list --type DataFile` and capture full output
3. Run `seldon artifact list --type DataFile --state proposed` (or filter the above for proposed-state artifacts)
4. Note the count of proposed DataFile artifacts — this is your target

---

## Steps

### Step 1: Inventory proposed DataFiles

Run:
```bash
seldon artifact list --type DataFile
```

For each artifact in `proposed` state, record:
- Artifact ID
- Name
- File path
- Current documentation properties (if any)

Store as `{{result:DATAFILE_PROPOSED_COUNT:value}}`.

### Step 2: Verify each file exists on disk

For each proposed DataFile, confirm the file at its registered path actually exists. If a file is missing:
- Do NOT delete the artifact
- Log it as a warning: "DataFile [ID] references missing file: [path]"
- Skip to next artifact

### Step 3: Backfill documentation properties

For each proposed DataFile where the file exists, read the file and populate documentation properties:

- **description**: One sentence describing what data the file contains
- **format**: File format (csv, json, txt, etc.)
- **generated_by**: If obvious from filename or directory context, note which script produced it. If not obvious, leave blank and flag.

Use `seldon artifact update` to set properties. Discover property names from existing verified DataFile artifacts:
```bash
seldon artifact list --type DataFile --state verified --verbose
```

### Step 4: Check provenance links

For each DataFile, check if it has a `GENERATED_BY` or `COMPUTED_FROM` relationship to a Script artifact:
```bash
seldon link list --source [artifact-id]
seldon link list --target [artifact-id]
```

If no provenance link exists and the generating script is identifiable from context (e.g., `entropy-leibniz-v3/results_minimal.json` was generated by `entropy-leibniz-v3/entropy_gp_minimal.py`), create the link:
```bash
seldon link create --source [datafile-id] --target [script-id] --type GENERATED_BY
```

If the generating script is ambiguous, flag it but do not guess.

### Step 5: Promote to verified

For each DataFile that now has:
- A description
- A confirmed file on disk
- A provenance link (or a documented flag explaining why one can't be determined)

Promote state:
```bash
seldon artifact update [artifact-id] --state verified
```

### Step 6: Report

Create a summary of actions taken:
- Total proposed DataFiles found
- Files promoted to verified
- Files flagged (missing on disk, ambiguous provenance)
- Any errors encountered

seldon artifact list --type DataFile --state proposed

Expected: 0 proposed DataFile artifacts (or only those explicitly flagged as unresolvable).
