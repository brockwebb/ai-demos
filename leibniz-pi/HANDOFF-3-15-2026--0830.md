## What Needs Doing

### Experiment: Terminals × Population Heat Map

**Goal:** Produce a 2D grid showing discovery rate (out of 5 seeds) as a function of terminal count and population size. This characterizes the phase transition boundary and becomes a key figure in the preprint.

**Design:**
- Fitness: entropy (the stronger of the two)
- Terminal counts: 4, 6, 8, 10, 12, 15, 20
- Population sizes: 1000, 2000, 5000
- No injection, no ephemerals beyond the terminal count
- Terminal sets: for count N, use {k, 1, -1, 2} as the fixed set plus random integers from [-N, N] to fill to size N. OR: use {k} + integers {-floor(N/2)..floor(N/2)} for a clean symmetric range. The exact terminal composition at each level should be documented.
- 5 seeds per cell (42, 7, 137, 2718, 31415)
- MAX_SEED = 360s
- Same entropy fitness function as v3 minimal

**Output:** 7 terminal counts × 3 population sizes = 21 cells × 5 seeds = 105 runs. At 360s max per seed, worst case ~10.5 hours. Many seeds will early-stop, so actual time much less. Run populations in parallel where possible.

**Deliverable:** A results table and a heat map figure showing the discovery boundary. This is a CC task — write instructions in `CC_TASK_scaling_heatmap.md`.

**IMPORTANT design decision needed:** How to construct the terminal set at each size. Options:
- Option A: {k, 1, -1, 2} fixed + fill with integers to reach N. Guarantees Leibniz primitives always present.
- Option B: {k} + integers from a symmetric range. At N=4 this gives {k, -1, 0, 1} — missing 2. Leibniz not constructible without manufacturing 2.
- Option C: {k, 1, -1, 2} fixed + ephemerals from range(-N, N+1) as the expansion. Terminal count grows with ephemeral range.
- Recommend Option A for the heat map (controls for primitive availability, isolates the search space effect). Note this in the task.

## Theoretical Thread (for paper Discussion section)

Captured in RESEARCH_NOTES.md under "Second-Order Kinetics / Thermodynamic Connection." Key ideas:
- Leibniz error = 1/(2T+1) → second-order kinetics signature
- Entropy fitness = integrated second-order rate law
- GP convergence asks first-order question (is error shrinking?), entropy asks second-order question (is 1/error growing linearly?)
- Thermodynamic framing: Leibniz operates at steady-state far from equilibrium, constant rate of uncertainty dissipation
- Unexplored: efficiency fitness (bits per operation) as the most fundamental quantity

These are research notes, not verified claims. The paper Discussion section should present them carefully.
