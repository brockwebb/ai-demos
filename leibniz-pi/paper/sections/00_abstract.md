# Abstract

We investigate whether genetic programming can rediscover the Leibniz series for π/4, the alternating sum (-1)^k / (2k+1), from arithmetic primitives alone, without prior knowledge of the target formula. Using an log-precision fitness function that measures bits of precision gained per decade of summation terms, a minimal GP system reliably discovers Leibniz-equivalent expressions in all trials. Success requires restricting the terminal set to {k, 1, -1, 2}: {{result:entropy_minimal_5_5:value}}/5 seeds succeed under this constraint. Discovery fails sharply as the terminal set expands: at 15 terminals, success drops to {{result:entropy_stress_l1_0_5:value}}/5 despite the fitness function correctly ranking Leibniz as optimal when it is present.

We characterize this phase transition through a systematic grid of terminal counts and population sizes, identifying wrong-limit attractors as the primary failure mechanism. Wrong-limit attractors are structurally distinct series that converge to values near but not equal to π/4 within any finite evaluation horizon. The log-precision fitness corresponds to the integrated form of a second-order rate law, selecting for processes with constant precision gain rates. Leibniz gains exactly {{result:info_rate_3_32:value}} bits per decade, a constant rate across all scales. The difficulty of formula discovery is not in the fitness landscape but in the combinatorial coverage required to assemble correct building blocks from an expanding search space.
